# PRD-004: Honest Sample Report Page

**Status:** Implemented
**Created:** 2026-03-06
**Author:** Burhanuddin

---

## 1. The Emotion

A visitor clicks "View Sample Report" from the landing page. They've just read "Your code speaks. We verify it." — they're curious. This is the moment they decide if CredDev is real.

What they see today: a fabricated score dashboard — 82/100 overall, "Top 18-20% Globally", "Hire Confidence: 85%", "3% scam probability", "READY NOW" for FAANG. None of this exists in the actual product. The real report is a rigorous 9-page technical analysis that cross-references every claim against platform data with source citations. It says VERIFIED or UNVERIFIED. It flags risk. It's honest.

The fake dashboard promises a different product. A recruiter who tries CredDev expecting a score dashboard will receive a text report — and feel deceived. A developer who sees "82/100" will wonder how the number was calculated — because it wasn't.

The sample report page should make the visitor think: **"This is thorough. This is honest. I want one for myself."** Note: "This looks like every other SaaS dashboard with made-up numbers."

---

## 2. Mental Model (User Experience)

### What the visitor should experience

They land on the sample report page. Within 5 seconds they see:
- **Who this report is about** — name, date, data sources used
- **The verdict** — a concise summary of verified strengths, gaps, and positioning (pulled from Section 9 of the real report)
- **The evidence structure** — a scannable breakdown showing the report's key sections with real content: verified skills, risk flags, production signals, LeetCode depth

They scroll. Each section is presented as a clean, readable card — not a score dashboard. Where the real report says "VERIFIED", the page shows a green tag. Where it says "UNVERIFIED", it shows an amber tag. The source citations are present but subtle (collapsed or small text) — they signal rigor without cluttering the page.

At the bottom: a CTA to generate their own report + a "Download Full PDF" option for those who want the raw 9-page document.

### What gets removed from the current page

Everything. The current page is completely replaced. Specifically:

|----------------|-----------------|
| Overall Score: 82/100 | Doesn't exist in the real product |
| "Top 18-20% Globally" | No global percentile system exists |
| Problem Solving: 85/100, Engineering: 75/100, Consistency: 83/100 | Fabricated category scores |
| "Hire Confidence: 85%", "3% scam probability" | Not a real metric |

### What the new page shows (mapped from real report)

The real extensive report has 9 sections. The sample report page presents a curated subset — enough to demonstrate depth and honesty, not so much that it overwhelms:

| Real Report Section | How It Appears on Sample Page |
|--------------------|------------------------------|
| 1. Capability Identity | **Header area** — name, role, seniority, experience span, data sources |
| 9. Final Technical Positioning | **Verdict card** — the top-of-page summary. Verified strengths, significant gaps, overall signal quality. This is what recruiters and developers read first. |
| 2. Engineering Depth Analysis | **Verified Skills section** — languages and frameworks shown as VERIFIED/UNVERIFIED tags. This is the most visually striking part — green and amber tags showing what's proven vs claimed. |
| 3. Problem Solving Depth | **LeetCode section** — real numbers: problems solved, difficulty breakdown, acceptance rate, top topics. No fake scores — just the actual data. |
| 5. Production Readiness Signals | **Production Signals section** — which repos have CI, tests, Dockerfiles. Compact repo cards. |
| 8. Risk Flags | **Risk Flags section** — the honest part. Shows discrepancies, unverified claims. This is what makes CredDev credible — it doesn't hide problems. |

Sections not shown on the web page can be read in the full PDF download.

## 3. Requirements

### Must Have

- **No fabricated metrics.** Zero scores out of 100, zero percentiles, zero confidence percentages. Every number on the page must come from the real report PDF or real platform data.
- **VERIFIED / UNVERIFIED tags.** The signature CredDev feature — resume claims cross-checked against GitHub and LeetCode. Green for verified, amber for unverified. This is the most compelling visual on the page.
- **Verdict card at the top.** Pulled from Section 9 (Final Technical Positioning) of the real report. Concise summary: verified strengths, gaps, overall assessment. This is the hook.
- **Risk Flags section.** Shown honestly. Experience discrepancy, unverified revenue claims, missing Kotlin evidence. This proves CredDev is rigorous, not a vanity report.
- **Real LeetCode numbers.** 555 problems, Easy 205 / Medium 296 / Hard 54, acceptance rate 98.58%, top topics. No score — just data.
- **Real production signals.** Which repos have CI, tests, dependency files. Compact cards per repo.
- **Download Full PDF link.** For visitors who want the complete 9-page analysis. Link to the uploaded PDF or a static hosted version.
- **CTA at bottom.** "Generate Your Free Report" — same as landing page.
- **Mobile-first layout.** Everything readable on 375px. No hidden sections.
- **Dark theme consistent with landing page.** Same black/slate/purple aesthetic.

### Should Have

- **Source citations as collapsible detail.** Each section can optionally show the source paths (e.g., "Source: GitHub > repositories.nodes[name='cred-dev'].productionSignals"). Collapsed by default — visible on click/tap for those who want to verify rigor.
- **"This is a real report" badge/label.** Small indicator that this is actual CredDev output for a real developer, not a mockup.

### Won't Do (this PRD)

- Dynamic report pages (loading from DB) — this is still a static page for the sample
- Report sharing or social cards
- Multiple sample reports (Pradeep page will be addressed separately)
- Any changes to the report generation pipeline or PDF formatting

## 8. Risks

- **"Where are the scores?"** — Visitors accustomed to other platforms (e.g., HackerRank, Pluralsight) may expect numerical scores. Mitigated by: the VERIFIED/UNVERIFIED system is more honest and more differentiated. It's the product's unique value — lean into it.
- **Information density** — The real report is 9 pages. The web page must be scannable, not overwhelming. Mitigated by: showing curated sections (verdict, skills, LeetCode, production signals, risks), not the full report. Full depth available via PDF download.
- **Hardcoded content** — This is still a static page with hardcoded data from one specific report. When dynamic report pages are built (future), this page becomes redundant. Mitigated by: it's a stopgap. The effort is small (one file), and the trust benefit is immediate.
- **PDF hosting** — The PDF needs to be in `/public/reports/` which adds a file to the repo. Mitigated by: it's one small file (~21KB), and it's the founder's own report — no privacy concern.
