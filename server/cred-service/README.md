# CredDev Backend — Skill Intelligence Engine

FastAPI service that extracts developer data from multiple platforms and generates AI-powered credibility reports.

---

## Architecture

### Two-Phase Pipeline

```
Phase 1: EXTRACTION                    Phase 2: GENERATION
POST /api/v1/extract                   POST /api/v1/generate/{job_id}
        ↓                                      ↓
┌─────────────────┐                   ┌──────────────────────┐
│ GitHub Fetcher   │                   │ LLM Report Generator │
│ LeetCode Fetcher │  → raw_data DB   │ (3 reports with      │  → reports DB
│ LinkedIn Fetcher │                   │  guardrails system)  │  → PDF email
│ Resume Parser    │                   └──────────────────────┘
└─────────────────┘                            ↓
                                      SSE progress streaming
                                      GET /api/v1/generate/{job_id}/stream
```

### Services

```
server/cred-service/
├── app/
│   ├── main.py                 # FastAPI app, CORS, startup
│   ├── config.py               # Environment settings (pydantic-settings)
│   ├── database.py             # SQLAlchemy models (AnalysisJob, RawData, Report)
│   └── routes/
│       ├── extract.py          # POST /extract, GET /extract/{job_id}
│       ├── generate.py         # POST /generate/{job_id}, GET /generate/{job_id}
│       └── stream.py           # GET /generate/{job_id}/stream (SSE)
├── services/
│   ├── extraction.py           # Orchestrates platform fetchers
│   ├── github_fetcher.py       # GitHub API (repos, languages, activity)
│   ├── leetcode_fetcher.py     # LeetCode GraphQL API
│   ├── linkedin_fetcher.py     # LinkedIn URL/username capture
│   ├── resume_parser.py        # PDF resume text extraction (PyPDF2)
│   ├── raw_data_loader.py      # Loads raw_data from DB for generation
│   ├── report_generator.py     # LLM prompts + guardrails system
│   ├── report_storage.py       # Saves generated reports to DB
│   ├── email_service.py        # PDF generation + email delivery (Brevo/SMTP/Resend)
│   └── progress_manager.py     # In-memory SSE progress tracking
└── requirements.txt
```

## API Endpoints

### `GET /health`
Health check. Returns `{"status": "healthy"}`.

### `POST /api/v1/extract`
Start raw data extraction. Accepts multipart form data.

**Parameters** (all optional, at least one required):
| Field | Type | Description |
|-------|------|-------------|
| `github_url` | string | GitHub profile URL |
| `leetcode_url` | string | LeetCode profile URL |
| `linkedin_url` | string | LinkedIn profile URL |
| `resume` | file | PDF resume upload |
| `candidate_name` | string | Candidate's name (default: "Anonymous Candidate") |
| `candidate_email` | string | Email to receive PDF reports |

**Response:**
```json
{
  "job_id": "uuid",
  "status": "extracting",
  "message": "Extraction started. Poll GET /api/v1/extract/{job_id} for status."
}
```

### `GET /api/v1/extract/{job_id}`
Poll extraction status. Returns `status`: `pending` → `extracting` → `extracted` (or `failed`).

When `extracted`, includes the raw data payload.

### `POST /api/v1/generate/{job_id}`
Trigger report generation. Requires job status to be `extracted`, `failed`, or `completed`.

Sets job to `generating` and starts the background pipeline.

**Response:**
```json
{
  "job_id": "uuid",
  "status": "generating",
  "message": "Report generation started. Connect to SSE at GET /api/v1/generate/{job_id}/stream for live progress."
}
```

### `GET /api/v1/generate/{job_id}/stream`
**Server-Sent Events** endpoint for real-time progress.

```javascript
const es = new EventSource('/api/v1/generate/{job_id}/stream')
es.onmessage = (e) => {
  const data = JSON.parse(e.data)
  // data.stage: loading_data | generating_extensive | generating_developer |
  //             generating_recruiter | storing | sending_email | completed | failed
  // data.status: "completed" or "failed" on terminal events
  // data.percentage: 0-100
  // data.message: human-readable status
}
```

### `GET /api/v1/generate/{job_id}`
Get generation status and completed reports.

**Response (completed):**
```json
{
  "job_id": "uuid",
  "status": "completed",
  "candidate_name": "John Doe",
  "reports": {
    "raw_data": { ... },
    "reports": {
      "extensive_report": "markdown string",
      "developer_insight": "markdown string",
      "recruiter_insight": "markdown string"
    }
  }
}
```

## Status Flow

```
pending → extracting → extracted → generating → completed
                ↓                       ↓
              failed                  failed
```

- `pending` → job created, background task not yet started
- `extracting` → fetching data from platforms
- `extracted` → raw data ready, waiting for generation trigger
- `generating` → LLM generating reports
- `completed` → reports stored, email sent
- `failed` → error occurred (check `error_message`)

## Report Generation

### Guardrails System

Reports are generated by OpenAI GPT with a guardrails system that enforces:

1. **Citation rule** — extensive reports use inline citations `[Source: Platform]`; developer/recruiter reports use natural references
2. **Verification tiers** — claims classified as VERIFIED, PLAUSIBLE, or CLAIMED
3. **No assumptions** — missing data acknowledged, never fabricated
4. **Neutral language** — no promotional terms without evidence
5. **Disclaimer** — AI-generated analysis caveat included
6. **No hallucination** — only reference data present in the raw context
7. **Static output** — no conversational follow-ups or questions

### Three Report Types

| Report | Purpose | Citation Style |
|--------|---------|---------------|
| `extensive_report` | Comprehensive technical deep-dive | Inline citations `[Source: GitHub]` |
| `developer_insight` | Growth-focused career guidance | Natural references |
| `recruiter_insight` | Hiring decision intelligence | Natural references |

## Email Service

Three providers supported, auto-selected by the factory in `get_email_service()`:

| Priority | Provider | When Used | Config |
|----------|----------|-----------|--------|
| 1 | **Brevo** | Production (HTTP API, no SMTP ports needed) | `BREVO_API_KEY` |
| 2 | **Resend** | Deprecated, kept for future (needs custom domain) | `RESEND_API_KEY` |
| 3 | **SMTP** | Local development | `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD` |

Each report is converted to a styled PDF (reportlab) with CredDev branding and attached to the email.

## Environment Variables

```bash
# Database (use Supabase pooler URL for cloud deployments)
CRED_SERVICE_DATABASE_URL=postgresql://postgres.xxx:password@aws-0-region.pooler.supabase.com:6543/postgres

# APIs
GITHUB_TOKEN=ghp_xxx                    # Optional, improves GitHub rate limits
OPENAI_API_KEY=sk-xxx                   # Required for report generation

# Supabase (optional, for direct Supabase client usage)
CRED_SERVICE_SUPABASE_URL=https://xxx.supabase.co
CRED_SERVICE_SUPABASE_SERVICE_ROLE_KEY=eyJ...

# Email — Brevo (production, recommended)
BREVO_API_KEY=xkeysib-xxx
BREVO_FROM_EMAIL=your-verified@email.com
BREVO_FROM_NAME=CredDev

# Email — SMTP (local dev)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM_EMAIL=your@gmail.com
SMTP_FROM_NAME=CredDev

# Email — Resend (deprecated, for future use with custom domain)
RESEND_API_KEY=re_xxx
RESEND_FROM_EMAIL=CredDev <you@yourdomain.com>

# App
DEBUG=false
```

### Database URL notes

- Use the **Supabase connection pooler** URL for cloud deployments (Render, Railway) — direct `db.xxx.supabase.co` resolves to IPv6 which most cloud platforms can't reach
- The pooler URL format: `postgresql://postgres.PROJECT_REF:PASSWORD@aws-0-REGION.pooler.supabase.com:6543/postgres`
- Locally, the service falls back to SQLite if no PostgreSQL URL is configured

## Quick Start

```bash
cd server/cred-service
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# Create .env with at least DATABASE_URL and OPENAI_API_KEY
uvicorn app.main:app --reload --port 8000
```

### Test the pipeline

```bash
# Health check
curl http://localhost:8000/health

# Phase 1: Extract
curl -X POST http://localhost:8000/api/v1/extract \
  -F "github_url=https://github.com/username" \
  -F "candidate_name=Test User" \
  -F "candidate_email=test@example.com"

# Poll until status=extracted
curl http://localhost:8000/api/v1/extract/{job_id}

# Phase 2: Generate
curl -X POST http://localhost:8000/api/v1/generate/{job_id}

# Watch progress (or connect via SSE)
curl http://localhost:8000/api/v1/generate/{job_id}
```
